# TreeSpider Examples

## Basic Example
  
After installing TreeSpdier, if the project doesn't have a bundler, copy TreeSpider from the node_modules to the root of your project directory, then TreeSpider can be used in 2 ways, through ES6 import or through the browser script tag, first ES6 import.  
  
### Using TreeSpider in a project with no bundler

#### ES6 import
  
The ES6 bundles are in the `dist/es` directory, and the minified bundle will be used, example

```ts
import TreeSpider from "./treespider/dist/es/treeSpider.bundle.min.js"

const instance1 = new TreeSpider({
    targetContainer: "#container-1"
})
```

Then you will see something like this in the browser

export const TestTSpyder1 = React.memo(() => {

    React.useLayoutEffect(() => {
        new TreeSpider({
            targetContainer: "#test-cont-ex-1",
            width: "100%",
        })
    }, [])

    return (
        <div id="test-cont-ex-1" style={{marginBottom: 20}}></div>
    )
})

<TestTSpyder1 />
  
#### Browser - Script tag
  
The browser bundles are in the `dist/browser` directory, the browser bundles allow TreeSpider to be loaded through the script tag, example:  
  
```html
<div id="container-1"></div>

<script src="/treespider/dist/browser/treeSpdier.bundle.min.js"></script>

<script>
    const instance1 = new TreeSpider({
        targetContainer: "#container-1"
    })
</script>
```

Then you will see it display like the working example above

### Project with a bundler

If you're running TreeSpider in a project that has a bundler, then the process of initializing TreeSpider is simple as follows
  
```ts
import TreeSpider from "treespider"

const instance1 = new TreeSpider({
    targetContainer: "#container-1"
})
```

TreeSpider can work in both JavaScript applications and pure TypeScript applications because TreeSpider was built completely with TypeScript

## Using Custom background pattern
  
You can use custom SVG background with TreeSpider, to provide a custom SVG background the SVG must have been stringified, you can use the `new XMLSerializer().serializeToString()` method to serialize your SVG DOM element to string, or pass it directly to the `customBackground` option if it's already in the string format and it's not a DOM element, example

```ts
import TreeSpider from "treespider";

const custom_svg = document.createElementNS("http://www.w3.org/2000/svg", "svg")
// ... add other stuffs like rect, circle, paths and so on...

const custom_svg_bg = new XMLSerializer().serializeToString(custom_svg)

// OR if you already have the svg in a string format

const custom_svg_bg = '<svg xmlns="http://www.w3.org/2000/svg">...</svg>'

const instance1 = new TreeSpider({
    targetContainer: "#container-1",
    customBackground: custom_svg_bg
})

```

Then it will look something like this

Then to further customize our custom svg background, let's try to set the background size to `20px`, example

```ts
const instance1 = new TreeSpider({
    targetContainer: "#container-1",
    customBackground: custom_svg_bg,
    backgroundSize: "20px"
})
```

Then if you run that, it will look something like this
